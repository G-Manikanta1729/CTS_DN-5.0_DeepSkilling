## What is this project?

This hands-on exercise introduces **React Event Handling**, one of the most important concepts in React development.

According to the uploaded hands-on lab, you will create a React application named **eventexamplesapp** that demonstrates:

- React Events
- Event Handlers

The application consists of a counter, multiple buttons demonstrating different event handling techniques, and a Currency Convertor component. :contentReference[oaicite:0]{index=0}

---

## What problem does this project solve?

Modern web applications constantly interact with users.

Examples include:

- Clicking buttons
- Entering text
- Submitting forms
- Selecting options
- Uploading files

Without event handling, applications would simply display information and could not respond to user actions.

React Events allow applications to become interactive.

---

## Why is Event Handling Important?

Suppose you are developing an Online Banking application.

When the user clicks:

```text
Transfer Money
```

the application should:

- Validate the account
- Verify the balance
- Transfer the money
- Display a success message

All these actions occur because of **event handling**.

Similarly, in this project:

- Clicking **Increment** increases the counter.
- Clicking **Say Welcome** displays a welcome message.
- Clicking **Click on me** demonstrates a synthetic event.
- Clicking **Submit** converts Indian Rupees into Euro. :contentReference[oaicite:1]{index=1}

---

## Real-world Examples of React Events

React Events are used in almost every web application.

Examples include:

- Login Forms
- Registration Forms
- Shopping Cart Buttons
- Payment Systems
- Search Boxes
- Like Buttons
- Add to Cart
- Delete Buttons
- Online Examinations
- Banking Applications

Whenever a user interacts with a webpage, an event is generated.

---

## Why does React use Synthetic Events?

Browsers implement events differently.

Without React, developers would need to handle browser differences manually.

React solves this problem using **Synthetic Events**, providing a consistent event system that behaves the same across all supported browsers.

This makes event handling simpler, more reliable, and easier to maintain.

# 5. Background Concepts

---

# 5.1 What is an Event?

## Definition

An **event** is an action or occurrence that happens inside an application.

An event can be generated by:

- User interaction
- Browser actions
- Keyboard
- Mouse
- Forms

Examples:

- Clicking a button
- Typing in a textbox
- Moving the mouse
- Submitting a form

---

## Real-world Analogy

Imagine pressing the **doorbell**.

```text
Person Presses Bell

↓

Doorbell Rings

↓

Owner Opens Door
```

The button press is the **event**.

Similarly,

```text
User Clicks Button

↓

React Detects Event

↓

Function Executes
```

---

# 5.2 What is Event Handling?

## Definition

**Event Handling** is the process of executing specific code whenever an event occurs.

Example:

```text
Click Button

↓

Run Function

↓

Display Result
```

Without event handling, applications cannot respond to user interactions.

---

# 5.3 What are React Events?

React Events are events handled using React's event system.

Example

```jsx
<button onClick={handleClick}>
    Click Me
</button>
```

When the button is clicked,

React executes

```javascript
handleClick()
```

---

## Common React Events

| Event | Description |
|--------|-------------|
| `onClick` | Mouse click |
| `onChange` | Input value changes |
| `onSubmit` | Form submission |
| `onKeyDown` | Key pressed |
| `onMouseOver` | Mouse enters an element |
| `onFocus` | Input receives focus |
| `onBlur` | Input loses focus |

---

# 5.4 Event Handlers

## Definition

An **Event Handler** is a JavaScript function that executes when an event occurs.

Example

```jsx
function sayHello() {
    alert("Hello");
}

<button onClick={sayHello}>
    Click
</button>
```

Flow

```text
Button Click

↓

sayHello()

↓

Alert
```

---

# 5.5 Synthetic Events

## Definition

A **Synthetic Event** is React's wrapper around the browser's native event.

It provides a consistent interface for handling events across different browsers.

Instead of working directly with browser-specific events,

React uses Synthetic Events internally.

In this exercise, clicking the **"Click on me"** button demonstrates a synthetic click event by displaying the message **"I was clicked"**. :contentReference[oaicite:4]{index=4}

---

# 5.6 React Event Naming Convention

React event names follow the **camelCase** naming convention.

Examples:

| HTML | React |
|------|-------|
| onclick | `onClick` |
| onchange | `onChange` |
| onsubmit | `onSubmit` |
| onmouseover | `onMouseOver` |

React uses camelCase because JSX is based on JavaScript syntax.

---

# 5.9 The `this` Keyword

The uploaded hands-on lab mentions using the **`this` keyword** as one of the learning objectives. :contentReference[oaicite:7]{index=7}

In **Class Components**, `this` refers to the current object and is used to access state and methods.

Example:

```javascript
this.setState({
    count: this.state.count + 1
});
```

However, this implementation uses **Functional Components** with **React Hooks (`useState`)**, so the `this` keyword is **not required**.

---

# 5.10 Functional Components and `useState()`

A Functional Component is a JavaScript function that returns JSX.

Example

```jsx
function Counter() {

    return <h1>Hello</h1>;

}
```

To store changing values such as the counter,

React provides the `useState()` Hook.

Example

```javascript
const [count, setCount] = useState(0);
```

This allows the counter value to update automatically whenever the user clicks a button.

# 19. Best Practices

- Use descriptive function names.
- Keep event handlers short and focused.
- Use `useState()` for dynamic values.
- Use `preventDefault()` for forms.
- Separate UI into reusable components.
- Use arrow functions only when parameters are required.
- Keep business logic separate from JSX.
- Validate user input before processing.

# 20. Common Mistakes

### Mistake 1

Calling a function immediately.

Wrong

```jsx
onClick={sayHello()}
```

Correct

```jsx
onClick={sayHello}
```

---

### Mistake 2

Forgetting `preventDefault()`.

Wrong

```javascript
function handleSubmit(){

}
```

Correct

```javascript
function handleSubmit(event){

event.preventDefault();

}
```

---

### Mistake 3

Updating state directly.

Wrong

```javascript
count = count + 1;
```

Correct

```javascript
setCount(count + 1);
```

---

### Mistake 4

Passing parameters incorrectly.

Wrong

```jsx
onClick={sayWelcome("Welcome")}
```

Correct

```jsx
onClick={() => sayWelcome("Welcome")}
```

---

### Mistake 5

Not validating user input.

Always verify that numeric values are valid before performing calculations.

# 24. Frequently Asked Interview Questions

---

## Q1. What is an event in React?

**Answer**

An event is any user or browser action, such as clicking a button, typing in a textbox, or submitting a form, that triggers a response in a React application.

---

## Q2. What is React Event Handling?

**Answer**

React Event Handling is the process of responding to user interactions by executing JavaScript functions called event handlers.

---

## Q3. What is an Event Handler?

**Answer**

An Event Handler is a JavaScript function that executes when a specific event occurs.

Example:

```jsx
<button onClick={handleClick}>
    Click Me
</button>
```

---

## Q4. What is a Synthetic Event?

**Answer**

A Synthetic Event is React's wrapper around the browser's native event. It provides a consistent interface for handling events across different browsers.

---

## Q5. Why does React use Synthetic Events?

**Answer**

React uses Synthetic Events to:

- Ensure cross-browser compatibility.
- Provide a consistent API.
- Simplify event handling.

---

## Q6. What is the difference between HTML events and React events?

**Answer**

| HTML | React |
|------|-------|
| `onclick` | `onClick` |
| `onsubmit` | `onSubmit` |
| `onchange` | `onChange` |
| Lowercase | camelCase |

---

## Q7. Why does React use camelCase event names?

**Answer**

Because JSX is based on JavaScript, React follows JavaScript's camelCase naming convention.

---

## Q8. What is the purpose of `onClick`?

**Answer**

`onClick` executes a function whenever an element is clicked.

---

## Q9. What is `onSubmit`?

**Answer**

`onSubmit` is triggered when a form is submitted.

---

## Q10. Why do we use `preventDefault()`?

**Answer**

It prevents the browser's default form submission behavior, such as refreshing the page.

---

## Q11. What happens if `preventDefault()` is not used?

**Answer**

The browser refreshes the page after form submission, which may interrupt your React application.

---

## Q12. How do you pass arguments to an event handler?

**Answer**

Use an arrow function.

Example:

```jsx
onClick={() => sayWelcome("Welcome")}
```

---

## Q13. Can one event call multiple functions?

**Answer**

Yes.

Create another function that calls multiple methods.

Example:

```javascript
function increaseAndGreet(){

incrementValue();

sayHello();

}
```

After an event updates the state, React compares the Virtual DOM with the previous version and updates only the changed parts of the Real DOM.
